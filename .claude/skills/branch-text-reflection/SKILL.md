# 分岐テキスト反映 (compass)

原文 `public/originalBranchTexts/world<W>/<W>-<M>.txt` を、
`src/data/branch.ts` の `BRANCH_TEXTS["<W>-<M>"]` と `BRANCH_LAST_UPDATES["<W>-<M>"]` に反映する。

## 原文(txt)の構造

- タブインデントの階層テキスト。インデント0の行がノード名(分岐マス)、その下の行が条件。
- ノード名「出撃地点」はキー `"0"` に対応する。それ以外はノード名がそのままキー。
- 条件行は「上から順に評価」される前提の記述。文言は原文に忠実に転記し、意味の言い換え・条件の統合・順序の入れ替えをしない。
- 末尾の「最終更新: YYYY-MM-DD HH:MM」を `BRANCH_LAST_UPDATES` の該当キーに `"YYYY-MM-DD HH:MM"` 形式で転記する。

## 記法($記号)

branch.ts 冒頭のコメントが正。主に使うもの:

- `$e` : 改行。条件行の区切りはすべてこれ。
- `$i` : インデント1段。「〜の場合、」の下位行は行頭に `$i`、2段ネストは `$i$i`。
- `$bo`〜`$ob` : 太字。「分岐点係数」行は `$bo分岐点係数:N$ob` と太字にする(コロン有無は同エントリ内の既存表記に合わせる)。
- `$co`〜`$oc` : 赤字。「無視」などの注記に使う。
- `ACTIVE_PROGRESS`(`$sw`) : 能動分岐マスはこの定数を値にする。

## 反映ルール

1. 同 world・直近 world(61-x, 62-x など)の既存エントリを様式の手本にする。
2. txt に無いノードが branch.ts に既にある場合は消さずに残す(別ソース由来の情報のため)。値が `ACTIVE_PROGRESS` のノードも触らない。
3. 既存エントリにある `$co無視$oc` などシミュレータ都合の注記は維持する(「未開通は帰投」のような到達不能条件も原文どおり転記した上で、既存の注記を踏襲)。
4. 用語は原文に忠実に。ただし既存エントリで確立した表記があればそれに合わせる(例: 「機動部隊」→「空母機動部隊」)。
   - 索敵値の閾値が甲難度の値しか記載されていない場合、「甲: 」プレフィックスは付けず「索敵値N以上は〜」とだけ書く。乙丙丁など複数難度の閾値が記載されている場合のみ難度プレフィックスを付ける。
5. ノードの並び順は既存エントリの順を維持し、新規ノードは原文の出現順を参考に適切な位置へ挿入する。
6. 常に最小差分。関係ないエントリの整形・リファクタはしない。コメント追加禁止。テスト追加不要。
7. 反映後、型チェック(`npm run tsc`)が通ることを確認する。無関係な既存エラー(実装途中の calc_X_Y など)は報告のみで触らない。
8. ブラウザでの実画面確認は不要。ユーザーが自分で行う。devサーバーを起動・操作しない。
