ベータからの脱出に向けて、リリース前に潰す仕様と追加する仕様をまとめる。
中核の管理基盤（スプレッドシート + ワークスペース）は実装済みのため、本計画は
**「機能追加」より「リリース品質への引き上げ」**に主眼を置く。

## 2. ① 数値バインドのバグ修正 + 既存データの自己修復

### 2.1 問題（データ喪失バグ）

`schema.ts` の `multiplier` / `rate` は厳密な `number()`。一方 `App.vue` の count / rate
セルは `.number` の無い `v-model` で束縛されているため、**セルを編集した瞬間に値が
文字列 `"3"` になる**。

その結果、次の連鎖が起きる。

1. 文字列を含む `row_datas` が IndexedDB に保存される
2. 次回ロードで `SheetSchema.parse` が `number()` 検証に失敗
3. `sheet.ts` の `LOAD` の catch が発火し、**シート全体が空行に差し替わる**
4. ④が未実装のため **無通知で消える**

「周回数を打ち込んでリロードしたら中身が消える」経路が実在する。最優先で塞ぐ。

### 2.2 対応

- **前方修正**: `App.vue` の count / rate の input を `v-model.number` にする
  （現状 `src/App.vue` の該当2箇所）
- **後方修正（自己修復）**: `rowDataSchema` の数値フィールドを、文字列が来たら数値へ
  変換する寛容な型にする（valibot の `union` + `transform`、または coerce 相当）。
  これで既に `"3"` で保存済みのシートも corrupted 扱いにせずロードできる
- **NaN ガード**: `.number` で空欄が `NaN` になり得るため、`calculation.ts` の入口で
  数値化する（`Number(x) || 0` 相当）。合計・差分に `NaN` を混ぜない

### 2.3 対象ファイル

`src/App.vue` / `src/logics/schema.ts` / `src/logics/calculation.ts`

### 2.4 テスト

- 文字列 `multiplier` を含むシートがロードできる（corrupted にならない）
- 空欄セルを含む行で合計・差分に `NaN` が出ない
- 反映先: `tests/parse.test.ts` 拡張

## 3. ② 取り込みの堅牢化 + 失敗時 UX

### 3.1 問題

- `extract.ts` は固定ラベルの正規表現で、1件もマッチしないと `throw`
- `App.vue` の paste ハンドラの catch が `console.error` のみで、**ユーザーに何も出ない**
- 結果「貼っても何も起きない」が起き、コンセプトの核である取り込みが信頼できない

### 3.2 実サンプルの形式

kc3kai 出撃シミュの「Resource Per 〈条件〉」出力。1ブロックは次の3部構成。

```
旗艦撃沈率: 7.4% (撤退率: 25.6%, 旗艦撃沈以外: 67%)   ← ①条件率の行（％が2つ）
旗艦撃沈あたり                                        ← ②見出し行
燃料:	20073.243                                    ← ③資源行（区切りはタブ）
弾薬:	7583.81
鋼材:	19451.408
ボーキ:	9292.141
バケツ:	83.684
ダメコン: 0.137
```

英語版は同構造でラベルのみ差し替わる（`Flagship Sunk Rate` / `Avg Resource Per
Flagship Sunk` / `Fuel:` …）。条件は `旗艦撃沈 / S / A …`（`S率`, `A率` …）と変化する。

### 3.3 現行実装で壊れる箇所（サンプルで確定）

| 症状 | 原因 | 影響 |
|---|---|---|
| **英語で貼ると rate=0** | rate 正規表現が `旗艦撃沈率\|S率\|A以上\|B以上` のみ。`Flagship Sunk Rate` / `S Rate` を含まない | 資源は入るため `is_match_found=true`。エラーも出ず**サイレントに rate だけ欠落** |
| 日本語のランク率取りこぼし | 実ラベルは「〈条件〉率」形式。`A以上` / `B以上` は実ラベルと不一致の可能性 | `A率` / `B率` 等が拾えない |
| `撤退率` 誤取得のリスク | rate 行は百分率が2つ（`旗艦撃沈率: 7.4% (撤退率: 25.6%…)`）。ラベル総当たりを緩めると括弧内を拾う | rate に撤退率が入る |
| （問題なし）タブ区切り | `燃料:\t…`。現行 `:\s*` がタブを吸う | 変更不要 |

資源6項目（燃料/弾薬/鋼材/ボーキ/バケツ/ダメコン）の日英ラベルは現行 regex で一致。
`洋上補給 / Underway Replenishment` はサンプルに無いが任意項目として温存する。

### 3.4 対応

- **rate 抽出の作り直し**: ラベル総当たりをやめ、**行頭から最初の `(` までを対象に
  `率:` / `Rate:` の直後の百分率を1つ取る**。括弧内（`撤退率`, `S以外`, `Retreat`,
  `non-S`）を構造的に除外でき、条件ラベルの増減にも影響されない。
  日英どちらの `〈条件〉率` / `〈condition〉 Rate` も1つの規則で拾える
- **資源ラベルの辞書化**: 燃料/弾薬/鋼材/ボーキ/バケツ/ダメコン/洋上補給の日英ラベルを
  辞書として `extract.ts` から外出しし、区切りは `[:：]\s*`（全角コロン・タブ・空白）を許容
- **失敗時 UX**: 握り潰しをやめ、抽出0件時にトーストを出す
  （例:「シミュレータの結果テキストを貼り付けてください」）。
  部分成功時は「読み取れた項目 / 読み取れなかった項目」を軽く通知する

（任意・v1.0 スコープ外）②見出し行から条件名を取り、空の `row_name` を補完する案。

### 3.5 対象ファイル

`src/logics/extract.ts` / `src/App.vue`

### 3.6 テスト

以下4サンプルを固定データとして `tests/extract.test.ts` に追加する。

- 日本語・旗艦撃沈（rate=7.4、燃料=20073.243 …）
- 英語・Flagship Sunk（**rate=7.4 が拾えること＝回帰防止の要**）
- 日本語・S率（rate=0.4、燃料=411501.472 …）
- 英語・S Rate（rate=0.4）
- 追加検証: `撤退率` / `Retreat` を rate に取り込まないこと

## 4. ④ 壊れ / 移行失敗データの通知と退避データの取り出し

### 4.1 問題

仕様書 §5.3 は「検証失敗時は削除せず退避し、モーダルで通知し、退避データの
ダウンロード手段を提供する」と宣言しているが、通知経路が実装されていない。

- `workspace.ts` の `INITIALIZE` は `run_legacy_migration()` の戻り値を捨てており、
  `corrupted` の結果がユーザーに届かない
- ワークスペース / シートの parse 失敗時に `stash_corrupted_data` は呼ぶが、
  その後 `console.error` + 空データへフォールバックするだけで **通知されない**
- `ErrorView.vue` は本文表示のみで、退避データを取り出す手段がない

### 4.2 対応

- **配線**: `run_legacy_migration()` の `corrupted` を拾う。シート / ワークスペースの
  parse 失敗（`stash_corrupted_data` 実行）時にモーダル通知を出す
- **退避データのダウンロード**: 退避キー（`cost-manager:corrupted:*`）の中身を JSON で
  ダウンロードできるボタンを `ErrorView.vue` に追加する。modal ストアに退避キーを
  持たせて受け渡す

### 4.3 対象ファイル

`src/stores/workspace.ts` / `src/stores/sheet.ts` / `src/stores/index.ts` /
`src/components/ErrorView.vue`

### 4.4 テスト

- corrupted 経路で通知フラグが立ち、退避キーが渡ること
- 反映先: `tests/migration.test.ts` 拡張

## 5. ＋ シート複製機能

### 5.1 目的

仕様書 §8 の未決事項を解消する。イベント計画のバリエーション作成（同じ編成で
周回数だけ変える等）で実需要が高い。

### 5.2 仕様（決定）

| 項目 | 決定 |
|---|---|
| 引き継ぐ内容 | 行データ全体。シート名は「〈元の名前〉のコピー」 |
| 複製先の位置 | 元シートの直後（同じ親） |
| 承認ドメイン | ワークスペース共有のため対象外 |
| フォルダの複製 | v1.0 では **シートのみ**。フォルダ複製は対象外 |

### 5.3 対応

- `workspace.ts` に `DUPLICATE_SHEET(node_id)` を追加
  （新 ID 発行 → 元本体を `load_sheet` → 複製を `save_sheet` → ツリーの直後へ挿入）
- `tree.ts` に「対象ノードの直後へ挿入」するヘルパを用意（必要なら）
- `TreeNode.vue` に導線（右クリックメニュー、またはボタン）を追加

### 5.4 対象ファイル

`src/stores/workspace.ts` / `src/logics/tree.ts` / `src/components/TreeNode.vue`

### 5.5 テスト

- 複製後にノードが1件増え、本体が元と独立している（一方の編集が他方に及ばない）
- 複製が元シートの直後に入る
- 反映先: `tests/tree.test.ts` 拡張
