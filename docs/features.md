# 機能一覧（現状）

「資源マネージャ」（poyopoyo工具箱の一機能）の、現時点で実装済みの機能をまとめる。
コンセプトは **艦これ出撃シミュレータ（kc3kai）の演算結果に特化したスプレッドシート風
管理アプリ**。

- 技術スタック: Vue 3 + Vite + Pinia + valibot + lz-string
- データはブラウザ内（IndexedDB）にのみ保存。サーバー同期・アカウントは無い
- 対象ブランチ: `feature/workspace-tree`

> 関連ドキュメント: [ワークスペース機能 仕様書](./workspace-spec.md) / [正規版v1.0リリース計画](./release-plan-v1.md)

---

## 1. スプレッドシート（シート本体）

出撃計画を1行1エントリで管理する表。

- **列構成**（左から）
  | 列 | 内容 |
  |---|---|
  | Import | シミュ結果テキストの貼り付け口（→ §2） |
  | name | 行名（自由記述） |
  | url | シミュへのリンク（→ §2） |
  | count | 周回数。合計はこの回数を掛けた値になる |
  | rate(%) | 撃沈率・S率などの成功率 |
  | 燃料 / 弾薬 / 鋼材 / ボーキ / バケツ / ダメコン / 洋上補給 | 資源7項目 |
  | （末尾） | 行クリア（X） |
- **行の編集**: 各セルを直接入力。名前セルで Enter を押すと下の行へフォーカス移動
- **行の追加**: 「行を追加」ボタンで10行ずつ追加（新規シートの初期行数は80行）
- **行のクリア**: 各行末尾の X で1行を空にする
- **行の並べ替え**: 左端のハンドル（⋮⋮）をドラッグ＆ドロップ
- **上詰め**: 空行を除いて詰める

---

## 2. データ取り込み（Import）

コンセプトの中核。シミュの結果テキストを解析して資源・成功率を自動入力する。

- **貼り付け取り込み**: Import 列にシミュ結果テキストを貼ると、成功率と資源を自動抽出する
  - 日本語・英語の両ラベルに対応（`燃料/Fuel`、`旗艦撃沈率/Flagship Sunk Rate` など）
  - 成功率の行は括弧より前だけを見るため、`撤退率 / Retreat` を誤って拾わない
  - 区切りは全角/半角コロン・タブ・空白を許容
  - 洋上補給はシミュ出力に無いことが多く、任意項目として扱う（欠けても警告しない）
  - **読み取れない場合**: トーストで通知（サイレントに失敗しない）
  - **一部だけ読めた場合**: 未取得の項目をトーストで知らせる
- **URL 列**
  - アイコンをクリックで元のシミュを新しいタブで開く（承認済みドメインのみ即時。未承認は
    確認モーダルを挟む → §9）
  - 右クリックで URL をコピー / 削除
  - ホバーで URL をツールチップ表示

---

## 3. 合計と差分（sum / diff）

表の下部に固定表示される結果行。

- **sum**: 全行の合計（各行 count 倍を反映）
- **diff**: 2行だけ選択すると差分表示に切り替え可能（`下の行 − 上の行`）。増減で色分け
- **切り替え**: sum / diff ボタンで明示的にトグル（2行選択時のみ diff 可）
- **結果行の縦移動**: 結果行のハンドルをドラッグして表示位置を上下に動かせる

---

## 4. 行の選択

左端のハンドル（⋮⋮）で行を選択する。

| 操作 | 挙動 |
|---|---|
| クリック | その行だけを単独選択 |
| Ctrl（⌘）+ クリック | 選択に追加 / 解除（複数選択） |
| Shift + クリック | 直前の起点からクリックした行までを範囲選択 |
| 表の外をクリック | 選択を解除 |

選択数に上限は無い。diff の成立条件（2行）とは切り離されている。

---

## 5. 行のコピー＆ペースト（シート間）

OS クリップボードを経由しない、アプリ内クリップボード。**別のシートへも貼り付けられる**。

| 操作 | 挙動 |
|---|---|
| Ctrl + C | 選択行をコピー |
| Ctrl + X | 選択行を切り取り（元の行は空になる） |
| Ctrl + V | 選択行の先頭を起点に上書き貼り付け |
| Ctrl + Shift + V | 選択行の直後に挿入して貼り付け |

シートの行数を超える貼り付けは、超過分だけ行を自動追加する。
セルを編集中は通常の文字コピーになり、取り込みセルの貼り付けとも干渉しない。

---

## 6. Undo / Redo

- 対象は **アクティブシート内の行データ編集のみ**（ツリー操作は対象外 → §7 で補う）
- キーバインド: Ctrl + Z（取り消し） / Ctrl + Y・Ctrl + Shift + Z（やり直し）
- 履歴上限は50件。連続したセル入力は1件にまとめる
- シートを切り替えると履歴はリセット。履歴は永続化しない（メモリのみ）

---

## 7. ワークスペース（サイドバー）

複数の計画をフォルダとシートのツリーで管理する。

- **開閉**: `s` キー、または左上のシート一覧ボタン（☰）。左からオーバーレイでスライドイン
- **ツリー**: フォルダ（ネスト無制限）とシートを混在表示。アクティブなシートを強調
- **シート切り替え**: ツリー上のシートをクリック。本体は都度 IndexedDB から遅延ロード
- **操作**
  - 新規シート / 新規フォルダの作成
  - リネーム（ダブルクリック）
  - **シートの複製**（⧉）: 行データごと複製し、元の直後に「〈名前〉のコピー」を作る
  - ゴミ箱へ移動（🗑）
  - ドラッグ＆ドロップで並べ替え・フォルダへの出し入れ（上端/中央/下端で挿入位置が変わる）
    - フォルダを自身の子孫へ移す操作は拒否される
    - 空き領域へのドロップはルート直下、ゴミ箱見出しへのドロップはゴミ箱へ
- **ゴミ箱**: サイドバー最下部に常設
  - 復元（↩）でいつでも元に戻せる
  - 「ゴミ箱を空にする」で初めてシート本体を物理削除（確認あり・取り消し不可）
- **移動の取り消し**: 直前の移動はトーストの「元に戻す」から1手戻せる

> 既知の制約: 並べ替えは HTML5 のドラッグ＆ドロップを使うため、タッチデバイスでは動作しない。

---

## 8. 保存と永続化

- **保存先**: IndexedDB（DB名 `cost-manager`）。ワークスペース（ツリー・設定）とシート本体を
  別レコードで持つ
- **保存タイミング**: 編集はデバウンス保存（500ms）。シート切り替え時とタブを閉じる直前にも書き出す
- **旧データの移行**: 旧 localStorage 形式（単一計画）を起動時に自動移行。旧データは削除せず
  バックアップとして残す
- **壊れ / 移行失敗データの保護**: 読み込めないデータは削除せず退避し、モーダルで通知。
  退避データはファイルとして保存できる

---

## 9. 共有・入出力

- **共有URL の発行**: 今開いているシート1枚を lz-string で圧縮した共有URLにする。
  TinyURL で短縮し、失敗時は短縮なしURLにフォールバック
- **共有URL の取り込み**: 共有URLを開くと、既存を上書きせず**新しいシートとして**追加する
  （非破壊）。承認ドメインは既存へマージ
- **ファイル入出力**（ファイルメニュー）
  - このシートを書き出す（JSON）
  - このシートを書き出す（CSV）
  - 全シートを書き出す（JSON・ワークスペース全体）
  - ファイルから読み込む（シート単体はルート直下へ、ワークスペースはインポート用フォルダへ）
- **承認済みドメイン**: URL を開く際の許可リスト。既定は `kc3kai.github.io`。未承認ドメインは
  確認モーダルを挟む

## 10. その他の UI

- **操作説明**: ヘルプ（？）ボタンでショートカットや操作の一覧を表示
- **初期化**: 開いているシートの内容をすべて削除（確認あり）
- **トースト通知**: 各種操作の結果や「元に戻す」アクションを表示
- **工具箱ナビ**: ヘッダーから poyopoyo工具箱の他ツール（羅針盤シミュ、資料室、ターミナル、
  任務手帳、改修効率計算機、入渠計算機、涼月タイマー）へ移動
- **フッター**: バグ報告・要望（お題箱）、作者 SNS へのリンク

## 付録A. キーボードショートカット

| キー | 機能 | 条件 |
|---|---|---|
| `s` | サイドバー開閉 | セル編集中を除く |
| Ctrl + C / X | 行のコピー / 切り取り | 行選択中 |
| Ctrl + V | 行の上書き貼り付け | 貼り付け先を選択中 |
| Ctrl + Shift + V | 行の挿入貼り付け | 同上 |
| Ctrl + Z | 取り消し | シート内編集 |
| Ctrl + Y / Ctrl + Shift + Z | やり直し | シート内編集 |
| Enter（name セル） | 下の行の name へフォーカス移動 | 変換確定を除く |

## 付録B. データモデル概要

- **Workspace**: ツリー（`nodes`）、承認済みドメイン、アクティブシートID。シート本体は含めない
- **ノード（TreeNode）**: フォルダ / シート。`parent_id` を持つフラットな集合として保持
  （ネスト無制限・移動が1レコード更新で済む）。ゴミ箱は実体のない `parent_id` として表現
- **Sheet**: シート本体（`row_datas`）。ツリーとは別レコードで遅延ロード
- **RowData**: 行1件（name / url / count / rate / 資源7項目）
- スキーマは valibot で検証。数値フィールドは文字列で保存された旧データも数値へ復元する
  （self-heal）
